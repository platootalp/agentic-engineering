## Key Decisions

1. **Layer-based grouping** over domain-modular вЂ” `models/`, `schemas/`, `services/`, `repositories/`, `api/v1/endpoints/` are global directories
2. **Preserve framework conventions** вЂ” FastAPI uses `middlewares/`, Express uses `middleware/`; each keeps its idiomatic naming
3. **Repository layer** вЂ” base template provides `base.py` / `base.repository.ts` abstract class only; concrete repositories added per entity
4. **Auth simplified** вЂ” auth-jwt capability generates minimal files: `core/security.py`, `core/dependencies.py`, `api/v1/endpoints/auth.py`, `schemas/auth.py`; no full users module
5. **Route aggregation** вЂ” `api/v1/router.py` is generated by base template with health only; capability modules use `force:true` to overwrite and append routes

## Capability Module Changes

| Module | Change |
|--------|--------|
| auth-jwt | Simplified: generates core/security, core/dependencies, endpoints/auth, schemas/auth only |
| db-sqlite/postgres/mysql | Same pattern: overwrite core/config, core/database, pyproject.toml, docker-compose |
| docker | Same pattern: overwrite Dockerfile, docker-compose |
| lint | Same pattern: overwrite linting configs |
