## 1. 产品背景与目标

### 1.1 痛点分析
- 求职者需要针对每个岗位调整简历，耗时耗力
- 手动匹配JD关键词容易遗漏
- 简历描述不够专业，无法突出与岗位的匹配度
- 投递多个岗位时，简历版本管理混乱

### 1.2 产品目标
打造"一键生成针对性简历"的AI工具，帮助求职者：
- 节省80%简历调整时间
- 提升简历与岗位匹配度
- 实现多岗位批量投递
- 建立个人经历库，长期复用

### 1.3 成功指标
- 用户留存率（7日/30日）> 40%
- 简历生成到导出完成率 > 70%
- 付费转化率 > 5%
- NPS评分 > 50

## 2. 产品架构

### 2.1 系统架构

三层架构：用户端 + 后端服务层 + 数据层

**用户端**:
- Chrome插件：JD提取、一键触发
- Web端：经历管理、简历编辑、数据分析
- 移动端（未来规划）

**后端服务层**:
- API Gateway：认证、限流
- 用户服务：注册/登录、订阅管理
- 经历服务：解析/存储、对话采集
- JD分析服务：关键词提取、要求解析
- 简历生成服务：AI匹配、模板渲染
- 文件服务：MinIO存储、下载链接

**数据层**:
- PostgreSQL：用户数据、经历数据、JD数据
- MinIO：文件存储、简历文件
- Redis：缓存、会话、任务队列

**AI服务层**:
- 云端模型：DeepSeek API、Moonshot API、SiliconFlow
- 本地模型：Llama 3、ChatGLM3

### 2.2 技术选型理由

| 组件 | 选型 | 理由 |
|------|------|------|
| 后端框架 | FastAPI | 高性能、自动生成API文档、Python生态 |
| 数据库 | PostgreSQL | 关系型数据、JSON支持、稳定性 |
| 文件存储 | MinIO | 对象存储、兼容S3、私有化部署 |
| 缓存 | Redis | 会话管理、任务队列、性能优化 |
| AI模型 | DeepSeek/Moonshot + 本地 | 成本可控、隐私保护、效果平衡 |
| 前端 | React | 组件化、生态丰富、性能优秀 |

## 3. 功能详细设计

### 3.1 插件端详细设计

#### 3.1.1 核心交互流程

1. 用户浏览Boss直聘页面
2. 插件自动检测页面类型（列表页/详情页）
3. 提取JD关键信息（职位名、公司、薪资、要求、描述）
4. 显示悬浮按钮（右下角"生成简历"）
5. 用户点击按钮，发送到服务端
6. 显示预览卡片（提取的关键信息）
7. 用户点击"去查看"，打开Web端

#### 3.1.2 插件界面状态

| 状态 | 视觉 | 行为 |
|------|------|------|
| 检测中 | 灰色旋转图标 | 不可点击 |
| 可提取 | 蓝色主按钮 | 可点击 |
| 提取成功 | 绿色对勾 | 显示预览卡片 |
| 提取失败 | 红色感叹号 | 显示错误信息 |

#### 3.1.3 插件权限配置

manifest.json关键配置：
- manifest_version: 3
- host_permissions: https://www.zhipin.com/*
- permissions: activeTab, storage, scripting
- content_scripts: 匹配zhipin.com所有页面

### 3.3 后端API设计

#### 3.3.1 核心接口

**认证相关**:
- POST /api/v1/auth/register - 用户注册
- POST /api/v1/auth/login - 用户登录
- POST /api/v1/auth/refresh - 刷新Token
- POST /api/v1/auth/forgot-password - 密码找回

**用户相关**:
- GET /api/v1/users/me - 获取当前用户信息
- PUT /api/v1/users/me - 更新用户信息
- GET /api/v1/users/subscription - 获取订阅信息

**经历相关**:
- POST /api/v1/experiences/education - 添加教育背景
- GET /api/v1/experiences/education - 获取教育背景列表
- PUT /api/v1/experiences/education/{id} - 更新教育背景
- DELETE /api/v1/experiences/education/{id} - 删除教育背景
- （工作经历、项目经验、技能特长类似）
- POST /api/v1/experiences/parse-resume - 上传并解析简历

**JD相关**:
- POST /api/v1/jd/analyze - 分析JD（插件调用）
- GET /api/v1/jd - 获取JD列表
- GET /api/v1/jd/{id} - 获取JD详情
- PUT /api/v1/jd/{id}/status - 更新JD状态
- DELETE /api/v1/jd/{id} - 删除JD

**简历相关**:
- POST /api/v1/resumes/generate - 生成简历
- GET /api/v1/resumes - 获取简历列表
- GET /api/v1/resumes/{id} - 获取简历详情
- PUT /api/v1/resumes/{id} - 更新简历
- POST /api/v1/resumes/{id}/export - 导出简历
- POST /api/v1/resumes/batch-generate - 批量生成

**对话相关**:
- POST /api/v1/chat/start - 开始对话采集
- POST /api/v1/chat/message - 发送消息
- GET /api/v1/chat/{session_id}/history - 获取对话历史

#### 3.3.2 响应格式

标准响应：
```json
{
  "success": true,
  "data": {},
  "message": "操作成功"
}
```

错误响应：
```json

  "success": false,
  "error": {
    "code": "INVALID_PARAMS",

  }
```

## 6. AI功能设计

### 6.1 JD分析流程

输入：JD文本
处理：
1. 提取职位名称、公司名称
2. 识别薪资范围、工作地点
3. 提取经验要求（年限）
4. 提取学历要求
5. 提取技能标签（关键词匹配）
6. 提取职位描述核心要求
输出：结构化JD数据

### 6.2 简历生成流程

输入：用户经历库 + JD分析结果 + 选定模板
处理：
1. 计算经历与JD的匹配度
2. 按匹配度排序经历（最相关优先）
3. 识别JD关键词
4. 使用AI重写经历描述（融入JD术语）
5. 调整描述顺序和强调重点
6. 渲染到选定模板
输出：优化后的简历内容

### 6.3 对话式经历采集

流程：
1. AI询问：请描述你最近的一份工作
2. 用户回答：我在XX公司做前端开发...
3. AI提取：公司名称、职位、时间
4. AI追问：你在那里主要做了什么项目？
5. 用户回答：...
6. 继续追问直到信息完整
7. 结构化保存到经历库

## 7. 安全与合规

### 7.1 安全措施

- 用户密码bcrypt加密
- JWT Token签名+过期机制
- API访问频率限制（按套餐）
- 用户数据严格隔离
- 简历文件加密存储
- HTTPS全站加密

### 7.2 合规要求

- 遵守《个人信息保护法》
- 用户数据可导出（JSON格式）
- 用户数据可彻底删除
- 明确告知AI使用范围
- 隐私政策透明
