## 1. 产品概述

### 1.1 产品定位
基于岗位JD + 用户个人经历，利用AI智能生成针对性优化简历的工具。

### 1.2 核心价值主张
- **省时**: 5分钟生成针对性简历，告别手动调整
- **精准**: AI分析JD关键词，自动匹配最优经历
- **专业**: 多模板选择，智能排版，一键导出
- **高效**: 批量生成多岗位版本，提升投递效率

### 1.3 目标用户
- 正在求职的个人用户
- 想跳槽的在职人员
- 应届生/实习生

## 3. 非功能需求

### 3.1 性能需求
- **NFR-001**: 插件JD提取响应时间 < 2秒
- **NFR-002**: 简历生成响应时间 < 10秒（单份）
- **NFR-003**: Web端页面加载时间 < 3秒
- **NFR-004**: 支持并发用户 1000+

### 3.2 安全需求
- **NFR-005**: 用户密码加密存储（bcrypt）
- **NFR-006**: JWT Token安全（签名+过期）
- **NFR-007**: 用户数据隔离（只能访问自己的数据）
- **NFR-008**: 简历文件加密存储
- **NFR-009**: API访问频率限制

### 3.3 可用性需求
- **NFR-010**: 系统可用性 99.5%
- **NFR-011**: 数据定期备份
- **NFR-012**: 故障恢复时间 < 30分钟

### 3.4 扩展性需求
- **NFR-013**: 支持新招聘网站扩展（脉脉、拉勾等）
- **NFR-014**: 支持新简历模板添加
- **NFR-015**: AI模型可切换（OpenAI/国产/本地）

## 4. 业务流程

### 4.1 首次使用流程
1. 用户注册账号
2. 上传现有简历或手动填写经历
3. 安装Chrome插件
4. 登录插件（与Web端账号关联）
5. 浏览Boss直聘，插件自动识别JD
6. 一键发送到服务端生成简历
7. Web端查看、编辑、导出简历

### 4.2 日常使用流程
1. 浏览职位列表/详情页
2. 插件识别JD并提取关键信息
3. 用户点击"生成简历"
4. 服务端分析JD + 匹配用户经历 + AI重写
5. 用户Web端查看生成的简历
6. 编辑调整（如需）
7. 导出投递

### 4.3 对话式补充经历流程
1. 用户在Web端选择"对话补充经历"
2. AI询问工作/项目细节
3. 用户回答
4. AI提取结构化信息
5. 确认后保存到经历库

## 7. 技术约束

### 7.1 技术栈
- **后端**: Python + FastAPI + PostgreSQL + MinIO
- **前端**: React/Vue + TailwindCSS
- **插件**: Vanilla JS (Chrome Extension Manifest V3)
- **AI**: OpenAI API兼容的国产模型 + 本地模型

### 7.2 合规要求
- 遵守《个人信息保护法》
- 用户数据可导出、可删除

## 8. 风险分析

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| AI生成内容质量问题 | 高 | 半自动流程，用户审核后再使用 |
| 招聘网站DOM变更 | 高 | 插件需要定期更新选择器 |
| 用户隐私顾虑 | 中 | 本地模型选项，数据加密，透明政策 |
| AI服务成本超支 | 中 | 分级限制调用次数，本地模型分流 |
| 竞争产品出现 | 中 | 差异化功能（对话式采集、多版本管理） |
